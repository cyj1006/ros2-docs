# 学习 ROS2 前需要准备什么



前面几章我们已经完成了“认知和选型”：



- ROS2 是什么、能干什么、为什么是 ROS2  

- 为什么新手推荐 Ubuntu（版本与安装方式）



这一章我们要做一件更现实的事：



> **在写 ROS2 代码之前，你需要准备哪些基础？**



!!! success "本章目标"

&nbsp;   让你用最短时间补齐“刚好够用”的前置技能，  

&nbsp;   不要求你变成 Linux 大神，也不要求你精通 Python/C++。



---



## 5.1 你要先建立的正确心态



很多新手会一上来就想：



- 先装 ROS2

- 立刻跑例程

- 看到机器人动起来



这没错，但如果你完全不具备基础，很容易卡在：

- 命令行不会用

- 环境变量不理解

- 报错看不懂

- 不知道该搜什么



!!! tip "正确策略"

&nbsp;   **先补“最低可用基础”，再开始装 ROS2**。  

&nbsp;   你会发现学习速度反而更快。



---



## 5.2 Linux 命令行基础（必须）



ROS2 的开发与调试，离不开终端。  

你不需要学会所有命令，只要先掌握下面这些就够了。



!!! info "最低命令清单（先会用）"

&nbsp;   - `pwd`：我现在在哪个目录  

&nbsp;   - `ls`：查看目录文件  

&nbsp;   - `cd`：进入目录  

&nbsp;   - `mkdir`：创建目录  

&nbsp;   - `cp / mv / rm`：复制/移动/删除  

&nbsp;   - `cat / less`：查看文件内容  

&nbsp;   - `grep`：在文本里搜索关键字  

&nbsp;   - `sudo`：管理员权限执行  

&nbsp;   - `apt update && apt install`：安装软件



!!! warning "新手高频坑"

&nbsp;   1) 不知道自己在哪个目录就乱执行命令  

&nbsp;   2) 把 `rm -rf` 当成普通删除（非常危险）



### 练习（5 分钟）

在终端完成以下操作：

1. 新建 `ros2_ws` 文件夹

2. 进入该文件夹

3. 新建 `notes.txt`

4. 用 `cat` 查看内容



---



## 5.3 Python 基础（强烈建议）



ROS2 的 Python 客户端库是 `rclpy`。  

对新手来说，用 Python 入门 ROS2 成本更低，反馈更快。



!!! tip "为什么新手推荐 Python"

&nbsp;   - 上手快、写得少  

&nbsp;   - 调试快（改完立刻运行）  

&nbsp;   - 先理解 ROS2 的概念，再去学 C++ 更轻松



你不需要精通 Python，只要先掌握这些：



!!! info "Python 最低知识清单"

&nbsp;   - 变量、列表/字典

&nbsp;   - 函数与参数

&nbsp;   - 类与对象（尤其是 `__init__`）

&nbsp;   - 继承（`class MyNode(Node):` 这种写法）

&nbsp;   - 字符串格式化（f-string）



### 练习（10 分钟）

写一个 Python 类，并在初始化时打印一句话，熟悉“类的结构”。



---



## 5.4 C++ 需要准备到什么程度？



如果你未来要做工程或高性能控制，C++ 很重要。  

但**入门阶段不建议同时硬啃 ROS2 + C++**。



!!! success "建议路线"

&nbsp;   - **入门用 Python**：先把节点/话题/服务跑通  

&nbsp;   - 后续再补 C++：把同样例程改写一遍



如果你一定要从 C++ 起步，至少要会：

- 基本语法（函数、类、头文件）

- 编译概念（CMake）

- 指针/引用基础



---



## 5.5 Git 与版本管理（推荐，但非常加分）



你做的是“工程”，不是“作业”。  

只要你想长期维护文档/代码，Git 就是必备技能。



!!! info "Git 最低命令清单"

&nbsp;   - `git clone`：下载仓库  

&nbsp;   - `git status`：当前修改状态  

&nbsp;   - `git add .`：暂存修改  

&nbsp;   - `git commit -m "msg"`：提交记录  

&nbsp;   - `git push`：推送到远程



!!! tip "学习 ROS2 时 Git 的意义"

&nbsp;   - 方便保存每一步的成功状态  

&nbsp;   - 出问题可以回滚  

&nbsp;   - 能把你的学习过程变成可展示作品



---



## 5.6 你需要准备哪些工具（建议装好）



下面这些工具能显著提升你的学习体验：



!!! info "建议工具清单"

&nbsp;   - VS Code（写代码 + 终端）

&nbsp;   - 浏览器（查官方文档/社区）

&nbsp;   - 终端（Ubuntu 默认即可）

&nbsp;   - 截图/录屏工具（记录你跑通的证据和过程）



!!! tip "强烈建议"

&nbsp;   养成“记录习惯”：命令、报错、解决方案都留下来。  

&nbsp;   这会让你后面写项目时效率翻倍。



---



## 5.7 推荐学习顺序（按周安排）



!!! success "推荐节奏（最稳）"

&nbsp;   **第 1 周：** Linux + Python + 工具（够用即可）  

&nbsp;   **第 2 周：** 安装 ROS2 + 跑通官方 demo  

&nbsp;   **第 3 周：** Node / Topic / Service / Param 全跑通  

&nbsp;   **第 4 周：** 做一个小项目（turtlesim 或自写 pub/sub）



!!! warning "不要一上来就做大项目"

&nbsp;   新手最大的问题不是“不会写”，而是“系统没跑通”。  

&nbsp;   先把最小闭环打通，再谈复杂项目。



---



## 本章小结



通过这一章，你应该已经清楚：



- 需要准备的不是“学很深”，而是“刚好够用”

- 终端命令是必须的

- Python 入门最推荐

- Git 会让你的学习变成可复用作品

- 按周推进，比随机学习更容易坚持

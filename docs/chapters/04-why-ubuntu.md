# 为什么选择 Ubuntu（系统与版本选择）



在真正开始安装 ROS2 之前，我们必须先解决一个**现实而关键的问题**：



> **我到底该用什么操作系统来学 ROS2？**



这个问题如果选错，后面你遇到的很多报错、环境问题、教程对不上，  

**都不是你的错，而是系统选择一开始就埋下了坑**。



---



## 4.1 结论先行：新手推荐的系统组合



!!! success "直接结论（新手必看）"

&nbsp;   **Ubuntu 22.04 + ROS2 Humble**  

&nbsp;   是目前对新手来说 **最稳定、教程最多、踩坑最少** 的组合。



如果你暂时记不住所有理由，**记住这一条就够了**。



---



## 4.2 为什么学习 ROS2 强烈推荐 Ubuntu



ROS2 是一个跨平台框架，理论上支持：



- Linux

- Windows

- macOS



但在**真实学习和开发场景中**，三者体验差异非常大。



---



### 4.2.1 Ubuntu 是 ROS 官方的“主战场”



!!! info "事实说明"

&nbsp;   ROS 官方文档、示例、教程，**几乎全部以 Ubuntu 为第一优先级**。



这意味着：



- 文档示例默认 Ubuntu

- 社区解答默认 Ubuntu

- 新功能优先在 Ubuntu 上测试



如果你用 Ubuntu，**90% 的教程可以照抄照做**。



---



### 4.2.2 Windows / macOS 的现实问题



#### Windows

- ROS2 支持 Windows，但：

&nbsp; - 教程少

&nbsp; - 第三方包不完整

&nbsp; - Gazebo / RViz 兼容性差

- 非常不适合新手作为主学习环境



#### macOS

- 更偏向开发测试

- 驱动和硬件支持较弱

- 社区资源更少



!!! warning "新手常见误区"

&nbsp;   “我电脑是 Windows，所以我要在 Windows 上学 ROS2”  

&nbsp;   这是**非常容易走弯路的想法**。



---



## 4.3 Ubuntu 版本与 ROS2 版本的对应关系



ROS2 的发行版与 Ubuntu 版本是**强绑定关系**。



!!! info "核心原则"

&nbsp;   **不要混搭版本，不要追新，不要随便升级系统。**



下面是最常见、也是最安全的组合：



| Ubuntu 版本 | 推荐 ROS2 版本 | 适合人群 |

|-----------|---------------|---------|

| 20.04 LTS | Foxy | 老项目 / 教材 |

| 22.04 LTS | **Humble** | **新手首选** |

| 24.04 LTS | Jazzy | 有经验者 |



---



## 4.4 为什么不推荐新手直接用 Ubuntu 24.04



很多新手会有一个直觉：



> **“版本越新，肯定越好。”**



但在 ROS2 学习中，这个直觉 **往往是错的**。



---



### 4.4.1 教程和生态尚未完全跟上



- 大量教程仍以 20.04 / 22.04 为主

- 搜到的问题答案可能不适用于 24.04

- 很多第三方包尚未完全适配



---



### 4.4.2 Gazebo / 仿真兼容性问题



!!! warning "真实踩坑点"

&nbsp;   新手最容易卡在 **仿真无法启动、模型加载失败、界面异常**。



这些问题往往不是你操作错了，而是：

- 版本组合不成熟

- 依赖尚未稳定



---



### 4.4.3 新手最重要的是“稳定”，不是“新”



!!! success "学习原则"

&nbsp;   学习阶段，**稳定 > 新版本 > 花哨功能**。



等你对 ROS2 足够熟悉，再尝试新系统，成本会低得多。



---



## 4.5 安装方式对比：我该怎么装 Ubuntu？



对于大多数新手来说，常见有三种方式。



---



### 4.5.1 虚拟机（不推荐长期使用）



**优点**

- 不影响原系统

- 安装简单



**缺点**

- 性能较差

- USB、显卡、仿真体验不佳



!!! info "适合谁"

&nbsp;   只是 **简单体验** ROS2 的人。



---



### 4.5.2 双系统（强烈推荐）



**优点**

- 原生性能

- 完整硬件支持

- 最接近真实开发环境



**缺点**

- 需要分区

- 初次安装有一点门槛



!!! success "推荐方案"

&nbsp;   **如果你认真学 ROS2，这是性价比最高的选择。**



---



### 4.5.3 WSL2（有条件可用）



**优点**

- 不用重装系统

- Linux 体验相对完整



**缺点**

- 图形界面、仿真配置复杂

- 新手容易卡在环境配置



!!! warning "新手提示"

&nbsp;   **不熟悉 Linux 的新手，不建议一开始就选 WSL2**。



---



## 4.6 我的最终建议（请认真看）



!!! success "最终推荐路线"

&nbsp;   1. 安装 **Ubuntu 22.04（双系统）**  

&nbsp;   2. 使用 **ROS2 Humble**  

&nbsp;   3. 按官方教程一步一步来  



这样做的结果是：

- 教程对得上

- 搜索能搜到答案

- 出问题更容易解决
